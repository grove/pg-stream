# v0.43.0 — Test Quality, Aggregate Correctness & Fuzz Automation

> **Full technical details:** [v0.43.0.md-full.md](v0.43.0.md-full.md)

**Status: Planned** | **Scope: Large**

> Replace fixed sleeps with state-polling helpers across the E2E suite, add
> differential SUM(CASE) and FULL JOIN aggregate property tests, harden WAL
> decoder SQL, update stale comments, and automate fuzz target execution in CI.

---

## What is this?

The assessment found 116 fixed sleeps in tests, missing differential coverage
for the SUM(CASE) fallback, coarse FULL JOIN aggregate testing, brittle WAL
decoder SQL construction, stale EC-06 comments, and no CI automation for fuzz
targets. v0.43.0 addresses all of these to improve test trust, reduce flakiness,
and lock down aggregate correctness.

---

## Replace fixed sleeps with state polling

Fixed sleeps make tests slow and flaky. v0.43.0 introduces polling helpers that
wait for explicit states — refresh history rows, CDC mode transitions, scheduler
tick watermarks, job status, NOTIFY signals, or catalog status changes — and
replaces the highest-impact sleeps across WAL CDC, scheduler, cascade, and
background worker test files.

---

## Aggregate correctness tests

`SUM(CASE ...)` has only a FULL-mode coverage example. The GROUP_RESCAN
fallback for this pattern can regress without test failure. v0.43.0 adds
differential E2E tests where UPDATE crosses the CASE condition threshold,
DELETE removes qualifying/non-qualifying rows, and INSERT adds both classes.

FULL JOIN + aggregate property tests compare DIFF vs FULL for nested FULL JOIN
aggregates across multi-cycle insert/update/delete sequences.

---

## WAL decoder SQL parameterization

WAL polling and decoded-change writes use dynamic SQL with manual escaping.
v0.43.0 parameterizes values that can be SPI parameters and centralizes strict
identifier constructors for slot names. Tests added for quotes, backslashes,
unicode, large text, nulls, and bytea-like payloads.

---

## Fuzz automation in CI

Fuzz targets exist for parser, cron, GUC, CDC, WAL, and DAG, but no workflow
runs them. v0.43.0 adds a scheduled fuzz smoke job with short time budgets per
target and corpus replay on PRs.

---

## Also in v0.43.0

- Stale EC-06 comments replaced with current keyless design notes
- Keyless multiset property test added

---

## Scope

v0.43.0 is a large release because replacing 116 sleeps, adding property-style
aggregate tests, parameterizing WAL decoder SQL, and setting up fuzz CI is
substantial infrastructure work. The payoff is dramatically improved test trust
and CI signal quality.
