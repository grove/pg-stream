# v0.42.0 — Documentation Truthfulness & Generated Reference Repair

> **Full technical details:** [v0.42.0.md-full.md](v0.42.0.md-full.md)

**Status: Planned** | **Scope: Large**

> Fix the broken SQL API and GUC catalogs, implement the missing
> `repair_stream_table` function, update the stale SQL reference signature,
> remove deprecated GUC references from tuning docs, and gate generated
> documentation freshness in CI.

---

## What is this?

The assessment found that documentation and implementation have significantly
diverged: the SQL API catalog misses functions, the GUC catalog shows
"registration pending" placeholders, `create_stream_table` docs omit newer
parameters, `repair_stream_table` is documented but not implemented, and
deprecated GUCs still appear in tuning guidance. v0.42.0 fixes all of these
and prevents recurrence through CI gates.

---

## Implement `repair_stream_table`

Backup/restore docs, the getting-started guide, and the error reference all
tell users to call `pgtrickle.repair_stream_table(name)` after restore — but
the function does not exist. v0.42.0 implements it: reinitialize storage, reset
frontiers, rebuild CDC triggers/buffers, verify dependencies, and add restore
E2E coverage.

---

## Fix catalog generation and gate in CI

The regex-based catalog generator misses private `#[pg_extern]` functions and
cannot link GUC statics to their later `GucRegistry::define_*` registrations.
v0.42.0 replaces it with Rust-aware extraction, regenerates both catalogs, and
adds a CI check that fails on any diff, pending registration, or missing known
function.

---

## SQL reference and docs freshness

The `create_stream_table` signature in docs omits `output_distribution_column`,
`temporal`, `storage_backend`, and other recent parameters. Deprecated GUCs
like `event_driven_wake` and `wake_debounce_ms` still appear as active tuning
knobs. Stale worker names (`pg_trickle.max_workers`,
`pg_trickle.max_parallel_refresh_workers`) appear in capacity planning docs.
Architecture docs reference old single-file modules.

v0.42.0 fixes all of these and adds `just docs-lint` plus CI gates to prevent
recurrence.

---

## Also in v0.42.0

- Deprecated GUCs moved to a dedicated compatibility appendix
- Architecture docs updated to current module layout
- RLS bypass warning added to Getting Started and Pre-Deployment docs

---

## Scope

v0.42.0 is a large release because it spans implementation (repair function),
tooling (catalog generator rewrite), documentation (6+ doc files), and CI
(docs-drift gate, docs-lint command). It is still assessment-driven hardening,
not new feature work.
