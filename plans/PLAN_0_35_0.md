# PLAN_0_35_0.md — v0.35.0 Implementation Order

**Milestone:** v0.35.0 — Citus: World-Class Distributed Source CDC
**Status:** Not started
**Last updated:** 2026-04-23

This document defines the recommended implementation order for all v0.35.0
roadmap items. See [roadmap/v0.35.0.md-full.md](../roadmap/v0.35.0.md-full.md)
for the full item list and exit criteria.

See also: [plans/ecosystem/PLAN_CITUS.md](ecosystem/PLAN_CITUS.md) §6 Phases
3, 4, 5, and 6.

**Prerequisite:** All v0.34.0 exit criteria must pass before beginning Phase 3
work. In particular, `SourceIdentifier`, `pgt_remote_slots` catalog, and the
stable-name object naming must be complete.

---

## Milestone Goals

1. **Phase 3 — Per-worker WAL slot CDC** — Poll `pg_logical_slot_get_changes()`
   on each Citus worker via `dblink`. New `pgtrickle.pgt_remote_slots`
   catalog. Changes decoded and written into the coordinator's
   `changes_{stable_name}` buffer with `origin_node`/`origin_lsn` columns.
2. **Phase 4 — Distributed ST apply path** — `create_stream_table(…,
   placement => 'distributed')`. DELETE + `INSERT … ON CONFLICT DO UPDATE`
   codegen. `__pgt_row_id` as distribution column. `reltuples` fix.
3. **Phase 5 — Coordination** — `pgtrickle.pgt_st_locks` catalog lock
   table. `pgtrickle.citus_status` observability view. Failure mode
   handling (unreachable worker, WAL recycled, slot missing, version
   mismatch).
4. **Phase 6 — Validation & docs** — Citus E2E docker-compose harness
   (1 coordinator + 2 workers), 10-scenario test matrix,
   `bench_remote_slot_poll`, `bench_distributed_apply`, SQL migration,
   `docs/integrations/citus.md`.

## Recommended Implementation Order

### Phase 3a — Catalog + Remote Connection (CDC-6, CDC-1)

Implement the lowest-risk, most-reviewable pieces first.

1. **CDC-6**: Add `pgtrickle.pgt_remote_slots` catalog table in
   `sql/pg_trickle--<prev>--<next>.sql`. Columns:
   `(id BIGSERIAL PK, pgt_id BIGINT, worker_id INT, worker_host TEXT,
   worker_port INT, slot_name TEXT, last_lsn PG_LSN, created_at TIMESTAMPTZ)`.
2. **CDC-1**: Add `poll_remote_slot(conn_str, slot_name, start_lsn) →
   Vec<DecodedChange>` to `src/wal_decoder.rs`. Uses `SPI::connect()` +
   `dblink(conn_str, 'SELECT * FROM pg_logical_slot_get_changes(…)')`.
   Wrap in a unit test with a mock that returns fixture rows.

### Phase 3b — Setup + REPLICA IDENTITY (CDC-2, CDC-3, CDC-8)

3. **CDC-2**: Extend `setup_cdc_for_source()` in `src/cdc.rs` /
   `src/api/mod.rs` to handle `placement == Distributed`:
   - `run_command_on_all_nodes(CREATE PUBLICATION …)`
   - `run_command_on_all_nodes(SELECT pg_create_logical_replication_slot(…))`
   - Insert rows into `pgt_remote_slots`.
4. **CDC-3**: Run REPLICA IDENTITY FULL check on each worker via
   `run_command_on_workers("SELECT relreplident FROM pg_class WHERE …")`.
   Enforce as pre-flight; emit `ERROR` if not FULL or INDEX.

### Phase 3c — Scheduler Polling (CDC-4, CDC-5)

5. **CDC-4**: In `src/scheduler.rs` tick: for sources with non-empty
   `pgt_remote_slots`, call `poll_remote_slot()` per worker row, batch
   decode, write into `changes_{stable_name}` with `origin_node`/`origin_lsn`.
   Add the two new buffer columns in MIG-1.
6. **CDC-5**: TRUNCATE action from WAL decoder already sets `action='T'`;
   extend to forward this to coordinator refresh logic. For DDL (column
   changes) on distributed sources, re-issue
   `run_command_on_all_nodes(ALTER PUBLICATION … SET TABLE …)`.

**Bench gate (BENCH-1):** Run `bench_remote_slot_poll` now. If throughput
< 10 k rows/s on loopback, escalate CDC-7 (libpq streaming) to P0 before
continuing. Document result.

### Phase 4 — Distributed ST Apply Path

7. **APPLY-5**: Extend `create_stream_table()` storage table creation to
   call `create_distributed_table(st_name, '__pgt_row_id')` when
   `placement = 'distributed'`. Add the `placement` parameter (APPLY-1).
8. **APPLY-3**: Add the DELETE + UPSERT codegen template in
   `src/refresh/codegen.rs`. Gate on `st_placement = 'distributed'`.
9. **APPLY-2 + APPLY-7**: Implement the placement auto-select heuristic
   (compare projected row count to `citus_reference_st_max_rows` GUC).
10. **APPLY-4**: Materialise delta to TEMP table before MERGE for reference
    STs where `EXPLAIN` shows a non-pushable plan.
11. **APPLY-6**: Fix `src/dag.rs` and `src/api/planner.rs` to sum
    `pg_dist_shard.shard_size` for distributed table row-count estimates.

### Phase 5 — Coordination

12. **COORD-1/COORD-2**: Add `pgt_st_locks` table (SQL migration). Wrap
    lock acquisition in `src/scheduler.rs` with a helper that uses the
    catalog table when `is_citus_loaded()`, falls back to advisory locks
    otherwise.
13. **COORD-3**: Implement `pgtrickle.citus_status` view in
    `src/api/cluster.rs`. Query `pgt_remote_slots` + `pgt_change_tracking`
    + a live `dblink` ping per worker.
14. **COORD-4/5/6**: Add failure-mode handling in the scheduler tick:
    - Unreachable: mark source as `paused`; emit Prometheus counter.
    - WAL recycled: detect `ERROR: requested WAL segment has already been
      removed`; trigger FULL refresh, log `WARNING`.
    - Slot missing: re-create via `run_command_on_workers`.
15. **COORD-7/8**: Pre-flight checks at `create_stream_table()` time.

### Phase 6 — Test Harness + Benchmarks + Docs

16. **TEST-1/TEST-2**: Write `tests/Dockerfile.e2e-citus` based on
    `citusdata/citus:13.0`. Write `tests/compose/citus.yml`
    (1 coordinator + 2 workers). Add helper in `tests/common/mod.rs`
    for `start_citus_cluster()`.
17. **TEST-3**: Implement the 10 E2E scenarios in
    `tests/e2e_citus_tests.rs`. Run sequentially
    (`--test-threads=1`) to avoid inter-test cluster state leakage.
18. **BENCH-2**: `benches/bench_distributed_apply` — compare
    DELETE+UPSERT vs MERGE on a 100M-row distributed ST. Document in
    `docs/BENCHMARK.md`.
19. **DOCS-1/2/3**: Write `docs/integrations/citus.md`, update
    `docs/ARCHITECTURE.md` multi-node diagram, update `INSTALL.md`
    Citus cluster section.

## Risk Register

| Risk | Likelihood | Mitigation |
|------|-----------|-----------|
| dblink throughput < 10 k rows/s | Medium | Run BENCH-1 early; if failed, escalate CDC-7 (libpq streaming) |
| Citus rejects MERGE on `__pgt_row_id`-distributed STs | Medium | DELETE+UPSERT is primary; MERGE only for reference STs |
| Slot fills WAL on worker if coordinator stops | High | Auto-drop slot at `citus_slot_max_lag_bytes`; Prometheus alert |
| Shard rebalance invalidates slots | Low | Hard error if topology hash changes; document rebalance procedure |
| Mixed extension versions across nodes | Medium | Pre-flight COORD-7 refuses to start if mismatch |

## Exit Criteria Checklist

- [ ] CDC-1: `poll_remote_slot()` decodes INSERT/UPDATE/DELETE/TRUNCATE from worker via dblink
- [ ] CDC-2: publication + slot created on every worker; `pgt_remote_slots` populated
- [ ] CDC-4: coordinator polls all workers; change buffer has correct `origin_node`/`origin_lsn`
- [ ] APPLY-1/APPLY-3: `placement => 'distributed'` end-to-end; DELETE+UPSERT apply correct
- [ ] COORD-3: `pgtrickle.citus_status` returns one row per (source, worker) with accurate lag
- [ ] TEST-3: all 10 E2E scenarios pass against 1+2 Citus cluster
- [ ] BENCH-1: throughput ≥ 50 k rows/s (dblink loopback); result documented
- [ ] Non-Citus benchmark suite: 0% regression (hard gate)
- [ ] DOCS-1: `docs/integrations/citus.md` complete
- [ ] `just test-upgrade-all` passes (v0.34.0 → v0.35.0)
- [ ] `just check-version-sync` passes
- [ ] `just lint` — zero warnings
