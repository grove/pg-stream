## Milestone Goals

v0.34.0 is the first of two Citus support releases. Its scope is entirely
additive and non-breaking:

1. **`src/citus.rs` detection module** — `is_citus_loaded()`, `placement()`,
   `worker_nodes()`, `shard_placements()`. Compile-time zero-cost when Citus
   is absent.
2. **`SourceIdentifier` struct** — carries `(oid, stable_name)` where
   `stable_name` is a 16-char hex derived from FNV-1a-64 of the
   schema-qualified table name. Identical on every Citus node.
3. **Catalog migration** — new nullable `source_stable_name`,
   `source_placement` columns on `pgt_change_tracking` and `pgt_dependencies`;
   `st_placement` on `pgt_stream_tables`.
4. **Rename all OID-keyed objects** — `changes_{oid}`, trigger function
   names, WAL slot names, and DVM placeholder tokens replaced with
   `{stable_name}` forms.
5. **Frontier rekey** — `Frontier.sources` rekeyed from OID string to
   `stable_name`; cross-source global LSN comparisons audited and removed.
6. **Per-node frontier column** — `frontier_per_node JSONB` for distributed
   sources (NULL for local/reference — used in v0.35.0).
7. **SQL migration** — `pg_trickle--<prev>--<next>.sql` renames existing
   objects in a single transaction; `just test-upgrade-all` must pass.

## Recommended Implementation Order

### Phase 1 — Detection + Naming Foundation (CITUS-1, CITUS-2, TEST-1, TEST-2)

Start with pure-Rust, zero-SQL additions that can be reviewed and tested
without a running database.

1. **CITUS-2**: Add `src/citus.rs` with just the `stable_hash()` function
   and `SourceIdentifier` struct. Export from `lib.rs`.
2. **TEST-1/TEST-2**: Write unit tests in `src/citus.rs` — known vectors,
   determinism across calls, no collision on common table names.
3. **CITUS-1**: Add `is_citus_loaded()`, `placement()`, `worker_nodes()`,
   `shard_placements()` to `src/citus.rs`. Guard all SPI calls with
   `if !is_citus_loaded() { return default; }`.

### Phase 2 — Catalog Migration (CITUS-3, CITUS-7)

4. **CITUS-3**: Write the SQL migration (column adds only — no renames yet).
   Run `just test-upgrade-all` to verify the column additions are safe.
5. **CITUS-7**: Add `frontier_per_node JSONB` in the same migration.

### Phase 3 — Object Renaming (CITUS-4, STAB-1)

This is the broadest change: touch `src/cdc.rs`, `src/wal_decoder.rs`,
`src/dvm/diff.rs`, `src/refresh/codegen.rs`, `src/dvm/operators/*`.

6. **STAB-1**: Add the OID-based fallback lookup first so that existing
   installations continue to work during the rename transition.
7. **CITUS-4**: Systematically replace every `{oid}` in object names.
   Replace in order: change buffer table, trigger function names, index
   names, WAL slot names, DVM placeholder tokens.

After each file change, run `just test-unit` to verify no regressions.

### Phase 4 — Frontier Cleanup (CITUS-5, CITUS-6)

8. **CITUS-5**: Rekey `Frontier.sources` in `src/version.rs`. Update all
   serialisation / deserialisation paths.
9. **CITUS-6**: Audit `src/refresh/codegen.rs` and `src/scheduler.rs` for
   `pg_current_wal_lsn()` cross-source comparisons. Replace each with the
   per-source watermark from `src/wal_decoder.rs`.

### Phase 5 — SQL Migration + Integration Tests (CITUS-8, TEST-3–6)

10. **CITUS-8**: Complete the SQL migration script — rename existing
    `changes_{oid}` tables, trigger functions, and WAL slots to the
    stable-hash form. Use a single transaction.
11. **TEST-6**: Write the v0.33.0 → v0.34.0 upgrade integration test.
    Run `just test-upgrade-all`.
12. **TEST-3/TEST-4/TEST-5**: Add integration tests for pg_dump round-trip,
    OID-stability, and multi-source frontier convergence.

### Phase 6 — Collision Detection + Docs (STAB-2, DOCS-1, DOCS-2)

13. **STAB-2**: Add hash-collision check at `create_stream_table()` time.
    Query `pgt_change_tracking` for an existing row with the same
    `source_stable_name` but a different `(schema, table)`.
14. **DOCS-1/DOCS-2**: Write changelog entry and upgrade guide section.
